Antes de empezar

Método de trabajo

Esta guía no está pensada para ser leída pasivamente. Cada ejercicio debe ser probado, alterado y comentado. Programar es aprender a producir variaciones controladas.

1. Ejecuta

Observa qué hace el ejemplo sin cambiar nada. Describe el resultado con tus propias palabras.

2. Modifica

Cambia números, colores, posiciones, textos o velocidades. Ejecuta otra vez y compara.

3. Explica

Escribe qué línea produjo el cambio. Si aparece un error, léelo como una pista.

Cómo usar los editores

Cada ejemplo tiene un editor de código y una vista previa. Modifica el código, pulsa Ejecutar cambios o usa Ctrl + Enter. El botón Restaurar vuelve al ejemplo original.

Capítulo 01

El lienzo

Todo en p5.js empieza con un espacio de dibujo. setup() se ejecuta una sola vez al inicio. draw() se ejecuta muchas veces por segundo. createCanvas() crea el lienzo.

Caja de herramientas: las tres piezas mínimas

Antes del primer ejercicio, necesitamos entender tres instrucciones básicas:

  • function setup(): bloque que se ejecuta una sola vez al iniciar.
  • createCanvas(ancho, alto): crea el espacio de dibujo en píxeles.
  • background(valor): pinta el fondo. Con un solo número usa escala de grises: 0 es negro y 255 es blanco.
function setup() {
  createCanvas(400, 300);
  background(255);
}
Ejemplo 01

Crear el primer lienzo

El programa crea un lienzo de 400 × 300 píxeles y lo pinta de blanco.

setup()createCanvas()background()
Retos
  • Cambia el tamaño del lienzo a 600 × 400.
  • Prueba background(0), background(120) y background(255).
  • Escribe un comentario que explique qué hace cada línea.

Caja de herramientas: color RGB

Cuando background() recibe tres números, los interpreta como rojo, verde y azul.

  • background(255, 0, 0): rojo.
  • background(0, 255, 0): verde.
  • background(0, 0, 255): azul.
  • Mezclar valores intermedios permite crear otros colores.

Capítulo 02

Coordenadas y formas básicas

El lienzo funciona como una cuadrícula. La coordenada x crece hacia la derecha. La coordenada y crece hacia abajo. Con esa lógica ubicamos figuras.

Caja de herramientas: coordenadas y figuras

Las figuras se dibujan indicando posiciones y medidas. El punto (0, 0) está en la esquina superior izquierda.

  • rect(x, y, ancho, alto): dibuja un rectángulo.
  • ellipse(x, y, ancho, alto): dibuja una elipse; si ancho y alto son iguales, parece un círculo.
  • line(x1, y1, x2, y2): dibuja una línea entre dos puntos.
  • point(x, y): dibuja un punto.
Ejemplo 03

Caja de herramientas: relleno, borde y superposición

Las formas se dibujan en orden. Lo último que dibujas queda encima de lo anterior.

  • fill(): define el color de relleno de las próximas figuras.
  • stroke(): define el color del borde.
  • noStroke(): elimina el borde.
  • Para tapar parcialmente una figura, dibuja otra encima con el color del fondo.

Un paisaje complejo puede armarse con instrucciones simples. La clave es pensar por capas: fondo, objetos lejanos, suelo y objetos cercanos.

  • triangle(x1, y1, x2, y2, x3, y3): dibuja un triángulo usando tres puntos.
  • width y height: guardan el ancho y alto del lienzo.
  • noLoop(): hace que draw() se ejecute una sola vez.

Capítulo 03

Color y composición

fill() controla el relleno. stroke() controla el borde. strokeWeight() controla el grosor. El cuarto valor en un color RGB controla la transparencia.

Caja de herramientas: transparencia

El cuarto valor de un color RGB se llama canal alfa. Controla la opacidad.

  • fill(255, 0, 0, 255): rojo completamente sólido.
  • fill(255, 0, 0, 120): rojo semitransparente.
  • fill(255, 0, 0, 0): rojo invisible.

Caja de herramientas: mapear valores

map() convierte un número de un rango a otro. Es una función central en programación creativa.

let nuevoValor = map(valor, rangoInicialMin, rangoInicialMax, rangoFinalMin, rangoFinalMax);

Capítulo 04

Texto y poesía visual

El texto también puede ser imagen. En p5.js podemos moverlo, deformarlo, hacerlo aparecer o convertirlo en una interfaz poética.

Caja de herramientas: escribir texto

El texto se dibuja como cualquier otra forma, pero con funciones tipográficas.

  • text("palabra", x, y): escribe una palabra o frase.
  • textSize(tamaño): cambia el tamaño de la letra.
  • textFont("Georgia"): cambia la fuente.
  • textAlign(CENTER, CENTER): cambia la alineación horizontal y vertical.

Variables

Una variable guarda un valor: una posición, un tamaño, una velocidad, un color, una palabra o un estado. Si una variable cambia dentro de draw(), aparece la animación.

Caja de herramientas: variables

  • Si una posición cambia en cada draw(), vemos movimiento.

Animación

Caja de herramientas: velocidad y rebote

  • x = x + velX: la posición cambia según la velocidad.
  • velX = velX * -1: invierte la dirección.
  • || significa “o”: una condición se cumple si ocurre una cosa u otra.

Capítulo 08

Condicionales

Un condicional permite que el programa tome decisiones. Si ocurre una condición, se ejecuta una acción; si no, se ejecuta otra.

Caja de herramientas: condicionales

Un condicional tiene esta estructura: si una condición se cumple, ejecuta un bloque de código.

if (condición) {
  // instrucciones
}

Las condiciones suelen comparar valores: mouseX > width / 2, x < 100, vidas === 0.

Caja de herramientas: if / else

else significa “si no”. Permite definir una alternativa cuando la condición principal no se cumple.

if (mouseX < width / 2) {
  background(192, 57, 43);

Capítulo 09

Bucles y patrones

Los bucles permiten repetir instrucciones sin escribirlas muchas veces. Son esenciales para patrones, grillas, textiles, visualización y arte generativo.

Caja de herramientas: bucles for

Un bucle for repite instrucciones. Tiene tres partes: inicio, condición y actualización.

for (let i = 0; i < 5; i++) {
  // esto se repite 5 veces
}
  • let i = 0: el contador empieza en 0.
  • i < 5: el bucle sigue mientras esto sea verdadero.
  • i++: aumenta el contador en 1.

Un bucle dentro de otro permite recorrer filas y columnas. El primer bucle puede controlar x y el segundo y.

for (let x = 0; x < width; x += 40) {
  for (let y = 0; y < height; y += 40) {
    ellipse(x, y, 20, 20);

Objetos y clases

Una clase es un molde para crear objetos. Cada objeto puede tener posición, velocidad, color y comportamiento propio.

Caja de herramientas: objetos y clases

Una clase es un molde. Sirve para crear muchos objetos con propiedades y comportamientos similares.

class Particula {
  constructor(x, y) {
    this.x = x;
    this.y = y;

  mover() {

  }
}
  • constructor(): define cómo nace el objeto.

  • new Particula(): crea una nueva partícula.

Capítulo 13

Azar y ruido

random() produce azar brusco. noise() produce variación suave, orgánica, parecida al movimiento de nubes, agua o paisajes.

Caja de herramientas: azar y ruido

random() salta de un valor a otro. noise() cambia suavemente. Por eso sirve para crear formas orgánicas.

  • random(0, 100): número aleatorio entre 0 y 100.
  • noise(t): valor suave entre 0 y 1.
  • t = t + 0.01: avanza lentamente por el ruido.

Cierre

Evaluación y rúbrica

Evalúa tanto el resultado visual como la comprensión del código. Lo importante es que el estudiante pueda explicar qué cambió, cómo lo cambió y por qué.

1. Código y estructura

  • Usa variables con nombres claros.
  • Organiza el código en bloques comprensibles.
  • Incluye comentarios útiles.

2. Interacción

  • Responde al mouse, teclado o clics.
  • Usa condicionales o estados.
  • El usuario entiende qué puede hacer.

3. Intención creativa

  • La pieza tiene una idea clara.
  • La forma visual refuerza el concepto.
  • El estudiante puede defender sus decisiones.

Preguntas para la bitácora final

¿Qué línea de código fue la más importante? ¿Qué error te enseñó más? ¿Dónde usaste variables? ¿Dónde usaste condicionales? ¿Qué parte de tu proyecto no habrías podido hacer sin bucles, arrays u objetos?
